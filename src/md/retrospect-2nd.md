# 두 번째 회고

## HTML 기초 다지기

이번 주에는 HTML의 기본적인 명령어들을 중심으로 수업이 진행되었다.

특히 기억에 남는 것은 두 가지였다.

---

### 1. Table 태그를 활용한 구조 만들기

이번 주에 배운 태그들 중에서 가장 흥미로웠던 건 **`<table>`** 요소였다.
단순히 데이터를 나열하는 용도라고만 생각했는데, 이를 통해 정보의 구조를 시각적으로 표현하는 방식이 꽤 논리적이고 재밌다는 걸 느꼈다.

- `<table>` : 표 전체를 감싸는 태그
- `<tr>` : 하나의 행
- `<td>` / `<th>` : 데이터 셀 / 제목 셀

구조가 명확하고 역할이 직관적이어서 배우는 과정이 즐거웠다.

---

### 2. Form 요소로 확장된 가능성

가장 재미있었던 부분은 **폼(form)** 관련 요소였다.
단순히 사용자 입력을 받는 도구가 아니라, 웹사이트가 사용자와 어떻게 상호작용하는지를 결정하는 핵심 요소라는 생각이 들었다.

특히 폼을 배우면서 자연스럽게 **웹 접근성**에 대해 많은 생각을 하게 되었다.

#### 예: 웹 접근성을 고려한 폼 요소들

- label과 input의 연결
- 스크린 리더 사용자를 위한 마크업
- placeholder vs aria-label
- 시각/청각/인지 장애 사용자의 접근 고려

폼은 정말 작은 요소 하나에도 여러 사용자의 상황을 고려해야 하기 때문에 책임감을 느꼈다.

## 웹 접근성과 나의 생각

사실 나는 예전부터 **소수자의 인권**이나 **사회적 약자**에 대해 관심이 많았다.
HTML의 폼 요소를 배우는 과정에서조차 이런 관심이 자연스럽게 떠올랐다.

> “단순히 작동하는 UI를 만드는 것보다,
> **누구나 사용할 수 있는 UI를 만드는 개발자가 되고 싶다.**”

기술이 널리 사용되는 만큼, 코드 한 줄도 모든 사용자를 고민하며 작성해야 한다고 느꼈다.

---

## 앞으로의 목표

- 접근성을 고려한 마크업을 기본값으로 삼기
- 다양한 사용자를 떠올리며 코드 작성하기
- 웹 표준 및 ARIA 속성 꾸준히 공부하기
- 작은 폼 하나라도 “누구나 사용할 수 있을까?” 생각해보기

## 마무리

이번 주는 단순한 HTML 기초 수업이 아니라,
**개발자로서 어떤 방향성을 가져야 하는가**를 고민하게 해준 뜻깊은 시간이었다.
앞으로도 “세심하게, 모두를 위한 코드를 작성하는 사람”이 되고 싶다.
