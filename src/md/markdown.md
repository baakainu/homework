# 마크다운 문법

## 1. Markdown이란?

- 경량 마크업 언어(Lightweight markup language)로, 사람이 읽기 쉬운 텍스트로 작성하면 렌더러가 HTML 등으로 변환합니다.
- 블로그, README, 과제, 문서화에 자주 사용됩니다.
- 대표 구현: CommonMark, GitHub Flavored Markdown(GFM) 등.

**장점**

- 배우기 쉽고 빠름
- 텍스트 기반이라 버전 관리(Git)에 적합
- 다양한 플랫폼에서 지원

---

## 2. 기본 문법

처음 과제로 Markdown을
공부해오라고 하셨을때,
어려워 보여서 당황했습니다.
하지만 패턴이 단순해서 금방 익숙해지고,
개발을 공부하기 전부터 보이던
문법들이 많아서 재밌었습니다.
아래는 자주 쓰이는 문법들입니다.

### 2.1 제목

**문법**:

```markdown
# 제목1

## 제목2

### 제목3
```

**설명**: `#`의 개수로 제목 레벨(1~6)을 표시합니다.

---

### 2.2 문단과 줄바꿈

**문법**:

- 문단은 빈 줄 한 줄로 구분합니다.
- 강제 줄바꿈은 줄 끝에 두 칸을 추가하거나 `<br>` 태그 사용.

```markdown
첫 번째 문단.

두 번째 문단.

줄바꿈하려면 두 칸를 넣 후 엔터  
다음 줄 계속
```

---

### 2.3 강조 (이탤릭 / 볼드 / 취소선)

**문법**:

```markdown
_기울임_ 또는 _기울임_
**굵게** 또는 **굵게**
~~취소선~~
```

**설명**: `*` 또는 `_`로 이탤릭, `**` 또는 `__`로 볼드, `~~`로 취소선.

---

### 2.4 리스트

**문법**:

- 순서 없는 리스트:

```markdown
- 항목1
  - 항목2

* 항목3
```

- 순서 있는 리스트:

```markdown
1. 첫째
2. 둘째
3. 셋째
```

- 들여쓰기(하위 항목): 스페이스 2~4칸

---

### 2.5 링크와 이미지

**문법**:

```markdown
[링크 텍스트](https://ex.com)
[타이틀 있는 링크](https://ex.com '타이틀')

![대체 텍스트](./img/pic.png)
![대체 텍스트](https://ex.com/pic.jpg '이미지 타이틀')
```

**설명**: 링크 괄호 `[]` 뒤에 URL을 `()`로 표기. 이미지 앞에 `!` 추가.

- **상대 경로**(예: `./img/pic.png`)는 저장소 내 이미지를 참조할 때 유용.

---

## 3. 자주 쓰이는 추가 기능 / 팁

- **HTML 태그 사용 가능**: 복잡한 레이아웃이나 스타일이 필요하면 직접 HTML을 삽입할 수 있다(`\<kbd>`, `\<span>`, `\<br>` 등).
- **이스케이프(특수문자 출력)**: `\*` 처럼 역슬래시로 특수문자 이스케이프.
- **파일 링크**: `[파일](./files/report.pdf)` 같은 상대경로로 파일 연결 가능.
- **이미지 크기 조절**(Markdown에는 없음): 대부분 환경은 HTML 태그로 조절 (`<img src="..." width="200">`).
